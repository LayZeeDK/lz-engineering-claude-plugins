### Gap 2 -- D-05 honesty gate (no owned row cites the book) -- GENUINE AUTOMATABLE GAP FOUND AND FILLED

**Requirement:** D-05 (17.1-CONTEXT.md) / explicit phase success criterion: "No owned Source cell
cites Test-Driven Development by Example (the book)." STRIDE threat T-17.1-03 (Tampering /
Repudiation: "a dishonest 'owned; oracle-verified' tag with no real full-text gate") names exactly
this failure mode.

**Adversarial check performed before writing any test:** read both `check-red-references.mjs` (the
topics arrays for `principle-backing.md` only assert `/oracle-verified/i` and `/no-oracle/i` appear
*somewhere* in the file) and `check-hygiene.mjs` (ASCII / email-allowlist / no-verbatim only,
no content-correctness checks). Neither script parses the map table rows, so neither could catch a
future edit that flips a row's Access-tier cell to `Owned` without also fixing its Source cell.
The phase's own Self-Check ("no 'Test-Driven Development by Example' remains ... git grep: 0 hits")
was a one-time manual grep at execution time, not a standing regression check. **This is a genuine
uncovered requirement, not a trivially-satisfied one.**

**Test generated:** `.claude/skills/lz-red-workspace/tools/lib/provenance-honesty.mjs`
(`findBookCitedAsOwned(text)` -- parses the pipe-table rows and flags any row where the Source cell
cites the book AND the Access-tier cell is `Owned`), wired into
`check-red-references.mjs` (extend-in-place, per that file's own documented "D-13 instrument-first,
extend-in-place, no sibling checker" convention -- no new checker script added to the battery).

**Behavioral self-test (can fail):** `.claude/skills/lz-red-workspace/tools/provenance-honesty.selftest.mjs`
-- plain `node:assert`, matching this workspace's existing assert-based style (no new test
framework introduced). Three assertions against in-memory fixtures (the real reference docs are
never touched):
1. A clean table (book cited but tagged `Unowned`) -> zero violations.
2. A REGRESSION fixture (book cited AND tagged `Owned`, simulating exactly the D-05 violation) ->
   the violating row IS caught by name.
3. Empty input -> zero violations, no crash.
