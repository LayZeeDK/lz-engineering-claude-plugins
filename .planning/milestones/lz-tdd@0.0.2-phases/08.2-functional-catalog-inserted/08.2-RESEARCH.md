### Count reconciliation (why 19, not ~17)

The board's ~17 (12 dissolves + 5 native) was explicitly indicative (D-02, D-13). The honest dedupe
yields 19 because:
- **Prototype gets a leaf (`structural-clone`), not a note.** Every gof/kerievsky/extra leaf must carry
  a resolvable `Functional alternative:` link to a LEAF (D-09/S13.4 #3). A README note has no leaf to
  link to, so a note-mapped GoF-23 pattern would break the uniform bidirectional check. +1 over a note.
- **`branded-type` is a distinct leaf.** Replace Primitive with Object + Replace Type Code with Class
  share a real `tsc --strict` idiom (branded types) that folds cleanly into no other leaf. +1.
- **`normalized-store` is D-10's explicit 6th native leaf** beyond the D-08 five (selector/lens folds
  INTO `lens-and-optics`, so it is not a separate 7th).

**Merge-down path to ~16 (flagged for discuss-phase, not recommended):** (a) merge
`thunk-and-lazy-value` + `memoization-and-interning` into one closure-captured-state leaf (Command +
Proxy + Flyweight); (b) demote `structural-clone` to a moot note and special-case Prototype in the
checker; (c) demote `branded-type` to a light note. Each trades checker uniformity / pedagogy for a
lower count. Recommendation: keep 19 -- the bidirectional check stays uniform and every GoF-23 resolves
to a leaf.

## The Cross-Link Contract (Deliverable 3)

Three link kinds, three enforcement scopes:

1. **`Functional alternative:` (forward, on 55 OO leaves).** Each gof (23) + kerievsky (27) + extra
   (5) leaf gains exactly ONE header line, placed right after the H1 (for gof/extra leaves, before
   `## Intent`; for kerievsky leaves, in the existing metadata header block alongside `Direction:` /
   `GoF pattern:`). Locked format:
   ```
   Functional alternative: [<idiom name>](../functional-catalog/<slug>.md#<anchor>)
   ```
   Resolves to a real functional-catalog leaf + anchor (via check-crossrefs, since gof/kerievsky/extra
   are already source files whose outbound links are resolution-checked).

2. **`Correspondence:` (backward, on 19 idiom leaves).** Each idiom leaf's header carries:
   ```
   Correspondence: dissolves-from | alternative-to  ->  [<Pattern>](../gof-catalog/<slug>.md#<anchor>), ...
   ```
   The enum value is exactly one of `dissolves-from` / `alternative-to`. The link(s) resolve to real
   gof/kerievsky/extra leaves (bidirectional set) OR fowler leaves (for `lens-and-optics`,
   `transducers`, `normalized-store`; resolution-only).

3. **Mutuality (the bidirectional guarantee, S13.4 #3).** For every OO leaf X with
   `Functional alternative -> idiom Y`, idiom Y's `Correspondence:` MUST link back to X. This is the
   inverse-of-style pairing the existing `check-crossrefs` already models. It is asserted forward-only
   (`forward-set subset of reverse-set`): an idiom leaf MAY carry ADDITIONAL Correspondence references
   (e.g. `functor-and-monad -> Iterator`) that are resolution-only and not required to be mutual,
   because Iterator's own `Functional alternative:` points to its dissolution leaf `generator`.

## Common Pitfalls

### Pitfall 1: DST-04 near-verbatim Intent trap
**What goes wrong:** the idiom one-liners ("a strategy is a variable whose value is a function", "a
one-method interface is just a function") reproduce Norvig/Wlaschin phrasing near-verbatim -- and the
research artifact QUOTES them, so copying from the artifact copies the quote.
**Why it happens:** canonical one-line idiom statements have a strong canonical wording (proven in
Phase 08.1: MEMORY "Pattern-leaf Intent near-verbatim DST-04 trap" -- 3 reviewers flagged it).
**How to avoid:** paraphrase every `## Idiom` line in original wording from the FIRST draft; do not
lift the artifact's quoted phrasing. Applicability/mechanical facts are lower-risk.
**Warning signs:** an `## Idiom` line that reads like a slide bullet; a quoted phrase inside the leaf.

### Pitfall 2: the N:1 residual-anchor cap
**What goes wrong:** an N:1 leaf's `## When each fits` grows to a paragraph per served pattern; C5
fails on the one-line cap, or the README row can't target a stable anchor.
**How to avoid:** author each served pattern as a `### <Pattern>` sub-heading + exactly ONE line; the
`###` produces the githubSlug anchor the README row targets.
**Warning signs:** a served pattern with two content lines; a README row linking to `<slug>.md` with
no `#anchor`.

### Pitfall 3: check-crossrefs RED-by-design mid-phase
**What goes wrong:** adding `Functional alternative:` links to the 55 OO leaves before the target
functional-catalog leaves exist makes check-crossrefs RED (unresolved link).
**Why it happens:** the same forward-reference RED-by-design seen in Phase 7 compose-links.
**How to avoid:** author the functional-catalog leaves in the SAME wave as (or a prior wave to) the
55 OO-leaf link additions; treat a transient check-crossrefs RED as expected, gate only at the phase
end.

### Pitfall 4: expression-problem guardrail misplacement
**What goes wrong:** the `Keep the OO form when:` line lands in the body instead of the header, or the
DU+fold leaf omits the "adding a new case edits every fold" flip.
**How to avoid:** the guardrail is a HEADER line (before `## Idiom`), and the DU+fold leaf's residual
lines carry the per-pattern expression-problem flip (Visitor/State/Interpreter/Composite each flip
differently -- new operation vs new case).

### Pitfall 5: Before->After that changes behavior
**What goes wrong:** the FP "After" is not observationally equivalent to the OO "Before"; tsc passes
(it checks types, not equivalence) so the error ships silently.
**How to avoid:** keep Examples pure with the same inputs -> outputs, ideally mirroring the linked
Fowler/Kerievsky refactoring's domain; the `mechanics: <link>, run tests between steps` line points at
the safe transformation. This is an authoring discipline, NOT a checker-enforceable property.
