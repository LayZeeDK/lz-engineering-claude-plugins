<objective>
Build the EVL-02 (RED-behavior) grading half: rewrite grade-run.mjs into the D-05-RUBRIC
per-dimension hybrid grader (deterministic phrase-set matchers for the six mechanical coach
dimensions + the LLM judge on exactly two judgment dimensions), borrowing the negation-aware
matcher, and author the RED-situation scenario set grounded in the shipped lz-red leaves. This
plan BUILDS assets only; it spawns NO subagent and runs NO metered benchmark (D-11).

Purpose: EVL-02 measures whether the coach recommends the correct next-test / assertion move
versus an unaided baseline. The one skill-specific script is the grader; everything else is reused
infrastructure. The grader must grade the observable dimensions deterministically (reusable across
iterations) and delegate only the two genuinely judgment-heavy dimensions, and it must not credit a
phrase the coach explicitly warned against (the Phase-11 grader-leniency class).

Output: the rewritten grade-run.mjs (skeleton kept, RED phrase-set + occursAffirmed + RUBRICS +
2-dimension judge + selfcheck with alignment + negation fixtures) and evals/evals.json (>=8
leaf-grounded RED scenarios). The behavior RUN (subagent fan-out + LLM judge) is gated to 20-03.

## Locked decisions honored
- **D-04:** Use the skill-creator test-case + benchmark posture -- evals/evals.json prompts, with-skill vs baseline (no-skill) runs, per-eval assertions, grade -> judge -> merge -> verify -> aggregate. Each scenario is code in a RED situation (a new behavior to test, or an existing test to critique/reshape), never a failing test to make pass (lz-tpp) and never passing code to restructure (lz-refactor). The subagent fan-out is an ORCHESTRATOR step (documented in 20-03), NOT an executor task.
- **D-05-RUBRIC:** Hybrid, decomposed into the shipped coach-procedure dimensions. Grade the mechanical/observable dimensions (selection; structure; assertion-target style named + matches the routed stance; fail-for-the-right-reason; handoff; coach-not-drive) DETERMINISTICALLY. Reserve the LLM judge for ONLY the two judgment-heavy dimensions: "is THIS genuinely the right next test" and "is the asserted target observable behavior not implementation detail". Each scenario pins a known-correct answer from a shipped lz-red reference example.
- **D-06:** Source scenarios PRIMARILY from the shipped lz-red reference examples so ground truth is pre-proven; cover a representative spread across the coach-procedure dimensions and the three stances, INCLUDING the required classify-first boundary scenario (a naive read routes to lz-tpp/lz-refactor but the correct move is a new RED test).
- **D-11 (HARD GATE):** Build the grader + scenario data; run NOTHING. No subagent spawn, no claude -p, no benchmark in this plan.
