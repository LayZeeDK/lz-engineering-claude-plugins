    Fill the leaf own-words. Author each anti-pattern as a section (or a table) carrying the contract
    fields Anti-pattern / Recognize by / Correction (observable-behavior fix). The six (D-11):
    1. Over-mocking / test-per-class rigidity (Cooper, OWNED -- author blind own-words): one spec per
       production class; a mock for every collaborator; asserting a save() was called. Correction:
       test behaviors through the public API / port; mock only at the module boundary (the outgoing
       command); trigger a new test on a new BEHAVIOR, not a new class.
    2. Testing private methods: reflection/casting to reach privates; tests break on internal rename.
       Correction: exercise privates through the public API; if that is hard, it is design feedback
       (extract a collaborator).
    3. Multiple unrelated assertions in one test: a name with "and"; asserts spanning behaviors.
       Correction: one concept per test; split (ties to STR-02, already filled Phase 16).
    4. A test that passes immediately (no red): green on first run before production code exists.
       Correction: see it fail first; a never-red test proves nothing. (Keep this as the ANTI-pattern;
       the LAW-02 fail-for-the-right-reason PROCEDURE is Phase 18 -- do not author the procedure here.)
    5. Snapshot-as-thinking: toMatchSnapshot as the primary assertion; blindly updating snapshots.
       Correction: assert the specific observable behavior; reserve snapshots for stable serializable
       output you actually review.
    6. Slow / order-dependent tests: shared mutable state; pass only in run order; hit real I/O.
       Correction: isolate state; push I/O to a shell; make each test independent + fast (F.I.R.S.T.).
