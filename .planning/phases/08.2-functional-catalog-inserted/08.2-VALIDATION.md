## Gap Audit (08.2-REVIEW findings WR-01 / WR-02)

The code review flagged two check-functional robustness findings. Assessed adversarially
(false-pass = coverage gap; false-fail = fail-safe, not a coverage gap):

### WR-01 - C4 did not enforce "EXACTLY one `Functional alternative:` line" - COVERAGE GAP, FILLED

- **Requirement facet:** FUN-04 ("each ... leaf carries a **one-line** `Functional alternative:` link") and the checker's own header contract ("EXACTLY one") were **not sampled** - C4 matched only the FIRST occurrence.
- **Proven false-pass:** appended a second, valid-resolving-but-semantically-wrong `Functional alternative:` line to `builder.md`; `check-functional` stayed **GREEN (exit 0)**. A wrong second link would slip past both C4 (first-only) and check-crossrefs (resolves, not semantic).
- **Fix (minimal, per review recommendation):** `check-functional.mjs` C4 now collects all matches with `matchAll` and asserts `length === 1` before validating the single line. No rewrite; `line` remains the first (now only) match, so all downstream logic is unchanged.
- **Post-fix probe:** same mutation now RED - `expected exactly one Functional alternative: line, found 2`. Clean tree stays GREEN.

### WR-02 - C1 `## Example` loose (`\b`) vs `sectionBody` exact - ACCEPTABLE RESIDUAL (fail-safe)

- **Assessment:** this is a **false-FAIL** path, not a false-pass. A descriptive heading like `## Example (Before -> After)` passes C1's `\b` but returns `null` from `sectionBody`, so `hasLiveTs` is false and the leaf FAILS with a misleading message. There is NO configuration where a missing/invalid Example passes - C6 always requires an exact `## Example` with a live ts fence.
- **Decision:** does NOT rise to a Nyquist coverage gap (the gate fails toward RED, the safe direction). Documented residual; the locked template uses a bare `## Example` in all 19 leaves, so there is no live failure. Left unmodified to stay minimal. Upgrade path if template drift ever introduces descriptive Example headings: tighten C1 to `/^##\s+Example\s*$/m` to agree with `sectionBody` and emit the correct diagnostic.

(Review INFO items IN-01 "55 count informational" and IN-02 "H1 fence-awareness" are covered by sibling checkers / match the established pattern; not coverage gaps.)
