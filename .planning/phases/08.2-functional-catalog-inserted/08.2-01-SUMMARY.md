## Deviations from Plan

None affecting scope. One deliberate faithful adaptation worth recording:

- **[Design adaptation, not a bug] `readmeRowsBySlug` regex broadened to accept an optional `#anchor`.** check-gof's copied regex (`/\]\(([a-z0-9][a-z0-9-]*)\.md\)/g`) matches only bare `slug.md` links. The N:1 README rows target `slug.md#pattern-anchor` (D-04/C5), so the C3 selector-mirror would never find rows for an N:1 leaf with the bare-only regex. Broadened to `/\]\(([a-z0-9][a-z0-9-]*)\.md(?:#[a-z0-9-]+)?\)/g`. This is the plan's own N:1 requirement, implemented -- not a departure. Documented inline in the file.

No leaf content, README rows, or `Functional alternative:` links were authored (those are waves 2-4, per the plan).
