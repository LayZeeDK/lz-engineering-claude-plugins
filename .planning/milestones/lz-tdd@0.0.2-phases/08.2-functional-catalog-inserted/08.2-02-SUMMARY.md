## Decisions Made

- **A4 nearest-idiom mapping** (Unify Interfaces -> first-class-function, Chain Constructors ->
  factory-function) -- see the dedicated section above.
- **Two ts fences per Example (Before OO + After idiom).** Each fence is extracted as its own module
  by extract-samples, so the OO `Shape`/`Formatter`/etc. and the FP redefinition of the same names
  never collide, and both halves are proven tsc --strict clean rather than only the After.
- **Correspondence carries the full served-leaf link set on one physical line.** C2 only requires one
  link of the right shape, but the plan (and the plan-06 mutuality target) want every served
  gof/kerievsky/extra leaf linked; the anchors use the shared `githubSlug` (note
  `Replace One/Many Distinctions with Composite` resolves to
  `#replace-onemany-distinctions-with-composite`, GitHub dropping the slash).
- **Residual lines are plain prose, no outbound links.** The `### <Pattern>` sub-headings are the
  anchor TARGETS the plan-05 README rows point to; keeping the single content line link-free honors
  the C5 one-line cap and adds no cross-ref resolution burden.
