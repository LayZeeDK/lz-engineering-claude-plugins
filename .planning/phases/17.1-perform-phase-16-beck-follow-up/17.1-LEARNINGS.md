# Phase 17.1 Learnings: perform-phase-16-beck-follow-up

## Decisions

### Content-rework allowed, not provenance-only (operator revision mid-phase)
The phase began as a provenance-ONLY tier upgrade (change attribution/tier text, never content). Mid-execution the operator revised the scope: content MAY be reworked to make a surface faithfully oracle-backed, with the per-surface goal being accurate-to-the-essay + own-words (no verbatim) + relevant to lz-red's RED purpose. Reverting to no-oracle is the fallback only when no free essay backs the concept even after a faithful rework.

**Rationale:** leaving a surface no-oracle solely to preserve pre-existing wording wastes a newly-provisioned owned source; the honest goal is accurate + owned where the source supports it.
**Source:** 17.1-CONTEXT.md (D-11)

### Only full-text FREE essays gate "owned"; book-summary entries stay no-oracle
A surface may be tagged `Owned; oracle-verified` only when gateable against real full-text (`Access: free`). `Access: book` entries in `.oracle/written-content/` are 1-2 sentence summaries and are NOT gateable; surfaces backed only by those stay no-oracle. An owned row must cite the free essay actually gated, never the summary-only book.

**Rationale:** a tier upgrade is honest only if the claim can be verified against source text.
**Source:** 17.1-CONTEXT.md (D-05), 17.1-01-SUMMARY.md

### Final tiers: 3 owned (Canon TDD), 3 no-oracle (book)
Owned = running test list, take one small step, assert-first (all Canon TDD). No-oracle = degenerate/starter case, triangulation, evident test data (Test-Driven Development by Example, held summary-only). Recommendation content unchanged.

**Rationale:** the oracle-reviewer gate + a follow-up oracle survey confirmed the three no-oracle surfaces are book concepts no free essay covers.
**Source:** 17.1-01-SUMMARY.md ("ORCHESTRATOR GATE RESOLVED"), 17.1-VERIFICATION.md

## Lessons

### The oracle-reviewer can split a bundled surface; finalize must re-split
Assert-first and evident-test-data were bundled into one blockquote clause + one `## Sources` entry, but are independent `principle-backing.md` rows. The gate resolved them differently (assert-first pass, evident-data blocked). A converge-to-clean finalize must be prepared to re-split a bundled provenance clause so each row carries its own tier. The plan-checker anticipated exactly this and the plan's orchestrator_gate was hardened for it before execution.

**Context:** clean-room re-gate loop; bundled-clause hazard.
**Source:** 17.1-01-SUMMARY.md, 17.1-VERIFICATION.md

### `phase.complete` false-flips a requirement from a bare ID token in the ROADMAP line
`phase.complete`'s requirements parser scans the ROADMAP phase "Requirements" line for req-ID tokens and marks them complete, IGNORING qualifier prose like "upholds but does NOT complete DST-03". It flipped DST-03 (a Phase-19, Pending requirement) to Complete. Fix: for requirements a phase does NOT complete, do not put the literal ID token in the ROADMAP Requirements line -- describe them without the ID (e.g., "the Phase-19 ship-hygiene constraints"). Then manually revert any wrong flip in REQUIREMENTS.md.

**Context:** a provenance-upgrade phase completes NO new requirement, yet tripped requirement-completion.
**Source:** 17.1-VERIFICATION.md (documentation-accuracy finding), phase.complete behavior

### A plausible essay-to-surface mapping can be wrong; survey before rework
"First One, Then Many" looked like it should back both the degenerate/starter-case and triangulation surfaces (its title implies one-then-many). The oracle survey showed it is production-code DESIGN succession (start from the single REAL case, defer the many-case), never discusses tests, and does not advocate the degenerate/empty/zero-first case -- so it backs neither surface honestly. Run the oracle survey (own-words facts) BEFORE reworking content, not after committing to a mapping.

**Context:** the content-rework path (per operator revision) required checking what each essay actually covers.
**Source:** oracle survey (own-words), 17.1-CONTEXT.md (D-11 outcome)

## Patterns

### Clean-room content-rework loop (source -> survey -> blind rework -> gate -> converge)
When re-tiering skill surfaces to owned after a source lands: (1) `oracle` agent surveys the essay for own-words facts (main context never reads prose); (2) blind author reworks the surface from those facts; (3) `oracle-reviewer` gates fidelity + eligibility; (4) converge-to-clean, reverting to no-oracle any surface the essay does not back. DST-04 stays intact because only own-words facts/verdicts cross back.

**When to use:** any owned-source upgrade or content rework against git-ignored `.oracle/` books/essays.
**Source:** 17.1-CONTEXT.md, 17.1-RESEARCH.md, this phase's gate

### Instrument-first honesty gate for content invariants
When a phase's success criterion is a content invariant (here: "no `Owned` row cites the summary-only book", D-05 / STRIDE T-17.1-03), add a table-anchored DETERMINISTIC check to the battery with a self-test that includes an injected regression fixture -- do not rely on a one-time manual grep at execution time. The nyquist auditor added `provenance-honesty.mjs` + a self-test and wired it into `check-red-references.mjs` (extend-in-place, no sibling checker).

**When to use:** any phase whose deliverable is a checkable content/attribution invariant with no runtime behavior.
**Source:** 17.1-VALIDATION.md (Nyquist Audit Trail)

### Orchestrator-owned oracle-reviewer gate (executor cannot spawn agents)
The gsd-executor has no Agent/Task tool, so the oracle-reviewer gate is planned as an ORCHESTRATOR post-execution step, not an executor task. The executor authors blind + runs the deterministic battery to GREEN and returns; the orchestrator then drives the gate and applies revert/re-split outcomes.

**When to use:** any GSD phase whose plan needs a subagent-run gate after the executor's deterministic work.
**Source:** 17.1-01-PLAN.md (orchestrator_gate), 17.1-01-SUMMARY.md

## Surprises

### The content-rework redirect investigated to the SAME no-oracle outcome
The operator's redirect (allow content change to reach owned) was expected to upgrade the three surfaces; instead the oracle survey (~93% confidence, essays read in full) showed degenerate-first, triangulation, and evident-data are genuinely Test-Driven Development by Example (book) concepts with no free-essay backing. The value was the INVESTIGATION and the honest confirmation, not a changed result. The real unlock (provisioning the book full-text) is a maintainer decision the operator declined.

**Impact:** reference docs stayed as the gate left them; the decision log now records WHY, grounded in a survey rather than a provenance-only rule.
**Source:** oracle survey, AskUserQuestion decision (keep no-oracle)

### A "no new behavior" prose phase still yielded a real automatable gap
The nyquist auditor's adversarial "assume uncovered until proven" stance found that the D-05 honesty invariant had zero standing regression protection (only a one-time manual grep), even though the battery asserted the tier TOKENS existed somewhere. It built a table-parsing check + self-test rather than rubber-stamping the prose phase as fully covered.

**Impact:** the battery now guards the honesty invariant on every future edit; the phase is genuinely nyquist-compliant, not nominally.
**Source:** 17.1-VALIDATION.md (Gap 2)

### Reverting an owned tag is never a false claim, which made the honesty call cheap
Because `no-oracle` (high-confidence core) is a weaker, always-true claim, reverting a surface from a provisionally-owned tag to no-oracle carries no correctness risk -- it only forgoes a stronger claim. This made the split-verdict resolution and the operator decision low-stakes, and is why auto-locking the earlier revert was safe.

**Impact:** the honesty-first bias (default to no-oracle when the essay does not clearly back the surface) is a safe, low-cost default for provenance work.
**Source:** oracle-reviewer gate reasoning, 17.1-CONTEXT.md
